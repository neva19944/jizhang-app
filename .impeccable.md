# 记账应用设计上下文

## Target Audience
- **主要用户**: 个人用户，需要记录日常收支
- **使用场景**: 日常消费记录、预算管理、财务分析
- **用户特征**: 希望简单快捷地记账，同时能看到清晰的财务 overview

## Use Cases
1. **快速记账**: 几秒钟内完成一笔收支记录
2. **查看账单**: 按日/月/年查看收支明细
3. **统计分析**: 了解消费习惯，优化支出
4. **账户管理**: 管理多个账户（现金、银行卡、支付宝等）
5. **预算控制**: 设置预算并追踪执行情况

## Brand Personality / Tone
- **感觉**:  trustworthy（可信赖）、clean（简洁）、professional（专业）、warm（温暖）
- **关键词**: 清晰、高效、愉悦、无压力
- **避免**: 过于花哨、令人焦虑的红色警告、复杂的金融术语

## Aesthetic Direction
- **设计方向**: Warm Minimalism（温暖极简主义）
- **色彩**: 使用温暖的蓝色/绿色作为主色调（传达信任和增长），避免传统的"金融红"
- **字体**: 清晰易读的无衬线字体，数字使用等宽字体增强可读性
- **空间**:  generous whitespace（充足留白），让数据呼吸
- **独特记忆点**: 数据可视化的微动画 + 温暖的渐变背景

## Technical Constraints
- 移动端优先（React + Vite + Capacitor）
- 需要支持 Android APK 打包
- 本地存储（localforage / IndexedDB）
- 无登录要求
